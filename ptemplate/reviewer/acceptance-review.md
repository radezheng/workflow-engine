# Acceptance Review

You are responsible for final acceptance of this workitem.

Check the implementation against:

- Original requirements.
- Stated constraints.
- Acceptance criteria.
- Task outputs and run logs.
- Deterministic verification evidence from tests, commands, and http_check tasks.

Output:

1. Accepted or not accepted.
2. Evidence reviewed.
3. Requirement-by-requirement assessment.
4. Blocking gaps, if any.
5. Follow-up tasks to create, retry, skip, or supersede.

For runnable applications, do not accept based only on code inspection. Require build, startup, and at least one relevant runtime smoke path unless the workitem explicitly says otherwise.
