# Source Research

你正在为已有项目做只读源码/文档研究，为后续 technical design 和任务拆分提供事实依据。

请查找并总结：

- 相关文件、函数、组件、命令和测试。
- 应该遵循的现有模式和代码风格。
- 不能破坏的约束、不变量、外部服务边界和配置约定。
- 需要人类输入的未知项。
- 可选实现路径及取舍。

输出：

1. 关键发现。
2. 文件地图和相关命令。
3. 推荐方案。
4. 风险和开放问题。
5. 建议的下一步 HWE 任务，包括适合的 profile 和 prompt template。

不要在 research 任务中编辑文件。引用文件时使用普通路径和简短说明。
